Academic Projects
Aug 2017 – Jul 2018 [Final Year Project]
Personalize e News Recommendation System

Abstract:
Information overload is one of the major problems exists in the e-news domain with the vast amount of content added by thousands of e-news web portals daily. So people feel it very difficult to read all the news articles and find the information what they want. It takes lot of time as well as man effort.

Hence, we aim to develop a “Personalized e-news recommendation system”. In this system we have identified four main modules; i.e. e-news extraction and classification, aggregation, summarization and recommendation.

For the e-news extraction module we have used a library called “Beautiful Soup” which extracts e-news from a set of pre-defined portals like BBC, CNN, Sunday Observer etc. The extracted raw data is stored in a HBase database which can manage all data in an effective way.

Then that raw data is pre-processed and then classified in to different groups. Then those classified news will be clustered in to different clusters which contain news articles about the same news from multiple sources together into a single place by the aggregation module.

Then the summarization module generates one single summary representing the key information of the news articles within a cluster.

The recommendation module gives personalized news recommendations for the users by tracking their past click events. So the user profiles also need to be stored in the HBase database. In this report we have included an introduction to the project, technologies chosen, review of other’s work, our approach to solve the problem, analysis & design and current implementation of the project.

Keywords:
Personalize, Extraction, Classification, Recommendation, Aggregation
